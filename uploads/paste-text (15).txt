ACT AS A SENIOR SAAS PRODUCT ENGINEER AND AI CONTENT PLATFORM ARCHITECT.

We are building the first AI-assisted content engine for PipingBox Community.

IMPORTANT:
Do not redesign the existing platform.
Do not break current Jobs, Dashboard, Companies, Applications, or Community features.

Build only the first MVP layer for AI content drafts.

==================================================
GOAL
====

Create a "Content Drafts" system where PipingBox can store AI-generated community post drafts before manual approval.

This is NOT automatic publishing yet.
This is human-reviewed content workflow.

==================================================
CONTENT CHANNELS
================

The system must support these suggested community channels:

1. Piping & Isometrics
2. Welding & QA/QC
3. Jobs & Projects
4. Tools & Calculators
5. Field Problems / Site Tips
6. Platform Updates
7. Ideas & Suggestions

Special purpose:

Platform Updates:
Used to publish product improvements, new features, bug fixes, release notes, platform progress, and visible updates so users feel there is an active team improving PipingBox.

Ideas & Suggestions:
Used for user feedback, feature requests, community proposals, and posts where PipingBox can later respond that a user suggestion has been implemented.

Example:
"Thanks for your suggestion. This week we implemented the feature you requested."

==================================================
STEP 1 — CREATE SUPABASE TABLE
==============================

Create table:

app_14da0f1941_ai_content_drafts

Columns:
id uuid primary key default gen_random_uuid()
created_at timestamptz default now()
title text
content text
source_url text
source_type text
suggested_channel text
language text
tags text
status text default 'draft'
created_by uuid
