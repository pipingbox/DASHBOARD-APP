FEATURE — Recruitment Messaging System (Company ↔ Worker)

IMPORTANT:
Do not redesign the platform.
Keep the same dark industrial UI style.

GOAL:
Add a professional recruitment messaging system between companies and workers.

This is NOT a public social chat.
This is a recruitment communication system.

==================================================
PHASE 1 SCOPE
=============

Allow messaging only between:

* company users
* workers

ONLY when:

* worker applied to company job
  OR
* company explicitly contacted/saved worker

==================================================
NEW FEATURE
===========

Add:

* Inbox / Messages page
* Conversation view
* Unread notifications
* Message counter badge

==================================================
DATABASE
========

Create table:
app_14da0f1941_messages

Fields:

* id
* conversation_id
* sender_user_id
* receiver_user_id
* application_id
* job_id
* company_user_id
* worker_user_id
* message
* read
* created_at

Create table:
app_14da0f1941_conversations

Fields:

* id
* company_user_id
* worker_user_id
* application_id
* job_id
* last_message
* last_message_at
* unread_company
* unread_worker
* created_at

==================================================
RULES
=====

Workers can:

* read own conversations
* send messages in own conversations

Companies can:

* read conversations linked to their jobs/applications
* send messages to candidates

Admins/jobs_moderators:

* full access

No public access.
